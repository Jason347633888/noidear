## 🎯 两个归档文件夹的区别

### design_modules/ - 设计文档归档

**内容**: PRD + LLD（产品需求文档 + 低层级设计文档）

包含：
- ✅ 业务需求描述
- ✅ 功能规格说明
- ✅ 数据模型设计（Prisma Schema）
- ✅ API 端点定义
- ✅ 业务规则（BR-xxx）
- ✅ 非功能性需求
- ✅ 验收标准
- ✅ UI/UX 设计规范

**用途**: 理解"要做什么"和"怎么设计"

---

### task_modules/ - 任务文档归档

**内容**: 详细开发任务分解

包含：
- ✅ 任务列表（按 Phase 分组）
- ✅ 每个任务的详细描述
- ✅ 工作量估算（小时）
- ✅ 优先级标注（P0/P1/P2）
- ✅ 依赖关系
- ✅ 验收标准（checkbox 格式）
- ✅ 技术要点
- ✅ 相关文件路径

**用途**: 指导"如何实施"和"任务拆解"

## 🚀 如何使用

### 1. 查阅设计文档

```bash
# 查看 MVP 模块 01 的设计文档
cat design_modules/01_三级文档管理.md

# 查看所有设计文档索引
cat design_modules/README.md
```

### 2. 查阅任务文档

```bash
# 查看 MVP 模块 01 的任务分解
cat task_modules/01_三级文档管理_TASKS.md

# 查看所有任务文档统计
cat task_modules/README.md
```

### 3. 对照阅读

开发某个模块时，建议同时打开：
1. **设计文档** (`design_modules/XX_模块名.md`) - 理解需求和设计
2. **任务文档** (`task_modules/XX_模块名_TASKS.md`) - 查看具体任务
