# 体系文件中心设计

## 目标

把当前重复的 `体系文件库` 和 `文档台账` 收敛成一个统一入口：`体系文件中心`。

`mybrain` 只作为本次初始化导入来源。导入完成后，`noidear` 系统里的 `Document`、`content_md`、附件、审批状态、版本、引用关系、复审日期、到期日期和审计记录才是后续唯一真实来源。后续编辑、换版、审批、归档、作废都不再同步回 `mybrain`。

## 现状问题

- `体系文件库` 和 `文档台账` 都在展示 `Document`，最后也都进入 `DocumentDetail`，用户无法判断哪个入口才是主入口。
- 导入的 Markdown 文件已经写入 `content_md`，但详情页没有直接渲染，也不能在线编辑。
- PDF 必须支持系统内实时查看，但当前预览能力不稳定；Word/Excel 可以先降级下载，PDF 不应该只下载。
- 已有 `DocumentReference`，但用户还不能在 Markdown 中写类似 Obsidian 的 `[[编号]]` / `[[标题]]`，也看不到反向引用。
- 供应商三证、供应商检测报告、产品外检报告、来料/批次检测报告等 PDF 有业务含义和有效期，不能只放在文档库里，否则供应商、产品、品质模块无法判断业务对象是否合规。

## 产品方向

建立一个主入口：`体系文件中心`。

中心内部基于同一批 `Document` 数据提供两个视图：

- `文件库`：面向日常查阅，按 `01/02/03/05/06`、文件类型、状态、部门、关键词查找当前受控文件。
- `台账`：面向文控管理，查看所有状态、版本、审批、归档、作废、复审、到期和审计字段。

旧的独立 `文档台账` 菜单应删除或重定向到 `体系文件中心` 的台账视图。

## 内容模型

`Document` 继续作为文件本体和文控生命周期的唯一主模型。

- Markdown 文件使用 `Document.content_md` 作为系统内正文。
- PDF、供应商证书、外检报告使用 `filePath`、`fileName`、`fileType`、`fileSize`、`external_expires_at` 和生命周期字段。
- `metadata.sourcePath` 和 `lineage_key` 只保留迁移来源痕迹，不驱动后续同步。

## 三类受控文件

体系文件中心管理三类文件：

1. **体系正文类 Markdown**：管理手册、程序文件、作业指导书、公司文件等，正文由系统维护。
2. **受控 PDF 附件**：外来标准、供应商证书、外检报告、产品测试报告、校准报告等证据文件。
3. **业务挂接类文件**：文件本体仍是 `Document`，但必须同时出现在供应商、产品、来料检验、批次、校准等业务模块中。

业务挂接类文件由 `Document` 负责存储、预览、版本、归档、作废、到期和台账；业务模块负责业务含义，例如供应商资质类型、报告结论、关联产品、关联批次、是否必需、是否满足准入或放行要求。

## Markdown 体验

Markdown 文件支持：

- 阅读模式：渲染 Markdown 正文。
- 编辑模式：Markdown 编辑器 + 实时预览。
- 保存到 `content_md`。
- 保存时产生版本记录或审计记录。
- 不写回 `mybrain` 文件系统。

第一版不做完整 Obsidian 克隆，只做文控场景需要的 Markdown 阅读、编辑、预览和引用。

## PDF / 附件体验

PDF 和证照类文件支持：

- 在系统详情页内实时预览 PDF。
- 无法预览时才降级下载。
- 展示有效期、复审日期、外部来源。
- 支持归档、作废、恢复、审批等统一文控生命周期。

Office 文件第一版可以继续作为附件管理，后续再考虑转 PDF 或 HTML 预览。Office 预览不阻塞 Markdown 和 PDF 闭环。

## 业务挂接与到期闭环

供应商证照、供应商检测报告、产品外检报告等需要形成完整闭环：

1. 用户从业务模块上传或换版 PDF，例如供应商详情、产品详情、品质/来料检验详情。
2. 上传创建或更新 `Document`。
3. 业务模块保存业务字段：
   - 供应商三证：营业执照、食品生产许可证、供应商资质证书。
   - 供应商报告：供应商检测报告、年度审核证据、第三方证书。
   - 产品/品质报告：产品外检报告、第三方检测报告、校准外检报告。
4. 用户可在业务模块和体系文件中心中实时预览 PDF。
5. 系统根据 `expires_at` 计算状态：
   - `valid`：有效期超过提醒窗口。
   - `expiring_soon`：即将到期。
   - `expired`：已过期。
6. 定时任务为责任人、责任角色或责任部门创建/更新 `TodoTask`。
7. 责任人上传新版本文件。
8. 旧文件归档或被新文件替代，业务记录指向当前有效文件，到期提醒待办关闭。

体系文件中心提供全局可见性和文控生命周期；供应商、产品、品质等模块提供业务判断和业务动作。

## 业务归属规则

- 供应商三证业务上属于供应商模块，文件本体属于 `Document`。
- 产品外检报告业务上属于产品或品质模块，PDF 文件本体属于 `Document`。
- 来料检验报告业务上属于 `IncomingInspection` 或 `MaterialLot`，PDF 文件本体属于 `Document`。
- 体系文件中心不替代供应商、产品、品质、来料检验模块，只提供统一文件底座、全局台账和预览能力。
- 业务模块不应重复实现文件存储、预览、到期、换版和归档逻辑。

## 轻量双向引用

Markdown 支持 wikilink：

```md
[[GRSS-CX-01]]
[[文件控制程序]]
[[原辅料验收标准]]
```

保存时后端解析 wikilink，并写入 `DocumentReference`：

- 优先按 `Document.number` 精确匹配。
- 其次按 `Document.title` 精确匹配。
- 匹配不到时创建未解析引用：`targetType='unresolved_document'`，`targetLabel` 为原始引用文本，`relationType='WIKILINK'`。

详情页展示：

- 当前文件引用了哪些文件。
- 哪些文件引用了当前文件。
- 哪些引用尚未解析。

不做关系图谱。

## 引用健康度

双向引用不能只显示“有引用”，还要显示引用是否还能成立。每条 `WIKILINK` 引用都需要有健康状态：

- `healthy`：正常引用。目标文件存在，且不是作废、归档或已被替代的旧版本。
- `dangling`：悬空引用。`[[引用文本]]` 没有匹配到任何体系文件。
- `invalid`：无效引用。引用曾经指向文件，但目标文件已删除、作废、归档，或不再允许作为当前受控依据使用。
- `conflict`：冲突引用。同一个引用文本匹配到多个候选文件，系统不能自动判断应该指向哪一个。
- `superseded`：被替代引用。引用指向旧文件，但旧文件已经有新的替代版本。

文件详情页展示当前文件的引用健康概览：

- 总引用数。
- 正常引用数。
- 悬空引用数。
- 无效引用数。
- 冲突引用数。
- 被替代引用数。

每个问题引用都要显示：

- 来源文件。
- 原始引用文本。
- 当前判断状态。
- 问题原因。
- 可跳转的目标文件或候选文件。

体系文件中心提供 `引用问题` 视图或筛选条件，用来全局查看：

- 哪些文件存在悬空引用。
- 哪些文件还引用了已作废、已归档或已删除的文件。
- 哪些文件还引用了被新版本替代的旧文件。
- 哪些引用文本命中了多个候选文件，需要人工确认。

这里仍然不做关系图谱。第一版目标是让文控人员能发现并修复问题引用，而不是做可视化探索。

## 导航

左侧菜单只保留一个主入口：

- `体系文件中心`

中心内部：

- 使用 Tab 或分段控制切换 `文件库` 和 `台账`。
- 点击文件进入统一详情/编辑页。
- 阅读确认、培训需求、文控健康度、审核覆盖、影响分析、审核链路等运营能力继续独立存在或挂在文控运营区域，不和文件查阅列表混在一起。
- 供应商、产品、品质模块可以跳转到 `DocumentDetail`，也可以嵌入 PDF 预览组件，但它们仍然是证照/报告的业务入口。

## 权限和生命周期

第一版复用现有文档权限和生命周期规则。

- 用户按现有规则查阅有效受控文件。
- Markdown 编辑权限沿用草稿/驳回文档可编辑逻辑。
- 生效文件不能直接改正文，应进入修订或换版流程。
- 归档、作废、恢复、审批继续复用现有生命周期。
- 到期提醒是执行型待办，不是审批任务。除非换版文件本身进入审批，否则提醒使用 `TodoTask`。
- 关闭换证/更新待办必须有新当前文件版本，或填写豁免/延期原因。

## 非目标

- 不做关系图谱。
- 不同步回 `mybrain`。
- 不做完整 Obsidian 工作区能力，例如 canvas、插件、日记、全库文件操作。
- 不新增平行文档主表。
- Office 在线转换不作为第一版阻塞项。
- 不把供应商、产品、品质模块强行合并进体系文件中心。

## 成功标准

- 用户只需要进入一个 `体系文件中心`，不再纠结 `体系文件库` 和 `文档台账`。
- Markdown 文件可以在系统内阅读、编辑、实时预览。
- PDF/证照/报告可以作为受控附件管理。
- PDF 可在系统内实时预览，无需下载。
- 供应商证照、产品/品质报告能挂接到所属业务对象，同时出现在体系文件中心台账。
- 即将到期和已过期的证照/报告会产生可执行提醒，并能通过换版/替代流程闭环。
- `[[编号]]` 和 `[[标题]]` 可产生正向引用和反向引用。
- 未解析、无效、冲突、被替代引用可见、可后续处理。
- 体系文件中心能全局列出引用问题，文控人员不用逐份文件打开排查。
- 已导入的 138 个受控文件继续保留在系统内，不再依赖文件系统同步。
