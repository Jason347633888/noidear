## 第 1 组：投料打料、配料、面浆、果酱过程

**确认状态：核心规则已确认。** 后续只需要在具体实现前再确认每个产品的固定工艺步骤和表单样式。

这组优先讨论，因为它涉及你刚才说的关键点：填表时可能还没有成品批次，且不同产品的表本质相似。

业务确认：**投料打料记录不是简单的“原料用量清单”，而是一个产品的搅拌工艺执行记录。**  

它记录的是：配料房、筛粉间、称油间等上游已经配好的料，进入打料/搅拌间以后，按工艺步骤加入到不同搅拌机里。这个过程有顺序，也可能有并行，例如一台搅拌机处理蛋黄，另一台搅拌机处理蛋白，后面再混合。

所以这组要分成两层理解：

1. **上游配料记录**
   - 配料房、筛粉间、称油间先各自完成称量、复核、准备。
   - 这些记录说明“料是怎么准备好的”。
   - 这部分和投料打料记录分开填。
   - 上游配料记录需要复核，不需要审批。
   - 上游配料记录里的标准重量也来自产品配方。
   - 实际称量原则上必须在产品允许误差范围内；超出误差范围时，必须走审批后才能进入下一步投料打料。
   - 只要和标准重量不一致，就需要记录差异原因。
   - 复核人必须从系统员工/用户中选择，不手写姓名。

2. **投料打料记录**
   - 在搅拌间记录“这些已经配好的料按什么顺序、进哪台搅拌机、搅拌多久、什么时候并行、什么时候合并”。
   - 它更像一个搅拌工艺步骤执行表，而不是单纯的配方行列表。
   - 搅拌机必须来自设备管理，不在表里手工新建设备。
   - 表里的重量对应各区域送上来的配料重量，用于确认实际进入搅拌过程的配料数量。
   - 这些重量应能反查到上游配料记录，例如小料房、筛粉间、称油间送来的某一份配料。
   - 投料打料时选择上游配料记录后，系统自动带出重量；如果实际进入搅拌的重量有差异，再记录差异。
   - 实际投入重量原则上必须在产品允许误差范围内；超出误差范围时，必须走审批后才能进入下一步。
   - 实际投入重量和上游送来重量或标准重量不一致时，需要记录差异原因。
   - 复核人必须从系统员工/用户中选择，不手写姓名。

业务确认：**产品投料打料表可以共用页面家族，但第一阶段重点是核对投料重量，不做复杂并行流程编排。**

补充确认：

- 每个产品的投料打料过程记录表确定下来后，通常不会频繁变化。
- 表里固然有工艺顺序、搅拌机、阶段，但填表时最关注的是投料重量对不对。
- 并行搅拌不用作为系统第一阶段重点，不需要做成可配置流程引擎。
- 系统只需要按该产品已确定的投料打料表样式生成填写项，让员工填写/核对重量。
- 投料打料记录需要审核/复核，不需要审批。
- 如果实际投入重量和标准重量不一致，需要记录差异原因。
- 是否能直接进入下一步取决于误差范围；第一阶段误差范围按产品统一设置，不做物料级误差配置。
- 超出误差范围时不能直接继续，必须记录差异原因并走审批，审批通过后才能进入下一步。
- 产品级误差范围配置在产品档案/配方维护处；投料打料记录只引用并保存当时的误差范围快照。

超差审批分配规则：

- 超差审批不写死给某一个固定人员。
- 使用现有统一审批模块发起审批实例，例如触发点可以叫“配料超差”或“投料打料超差”。
- 统一审批当前支持三种分配方式：指定用户、按角色、按部门 + 角色。
- 配料/投料打料超差第一阶段统一按“部门 + 角色”分配，默认给制造部 `leader` 审批。
- 人员变动时只维护制造部人员和 `leader` 角色，不需要改表单逻辑。
- 审批通过后，这条超差记录才能继续往下一步流转；审批不通过则停留在当前记录并保留原因、审批意见和审批人。

新产品规则：

- 新产品不是手工新建一张投料表。
- 先在产品档案/研发流程里建立产品、配方和打料/搅拌工艺步骤。
- 配方里维护会用到哪些原料、单批用量、单位、配料区域等信息。
- 打料工艺里可以维护固定表单结构、搅拌机、加入物料等信息；标准重量来自产品配方，不在投料打料表里单独维护。
- 该产品有当前有效配方和当前有效打料工艺后，系统就能生成对应的投料打料填写表。
- 以后配方新增或减少原料，只影响新填写记录；旧记录保留当时使用的配方快照。
- 以后投料打料表样式、搅拌机或标准重量变了，也只影响新填写记录；旧记录保留当时使用的表单快照。

## 第 2 组：过程参数、产量、计划、成品入库

初始建议：先保留填写入口；其中很多记录早期可能没有成品批次，后续再关联。

本组先按这个口径理解：

- 烤炉、蒸炉、面浆打发、包装用膜等过程记录，填写时不强制要求已经有成品批次。
- 这些记录先按日期、班次、线别、产品、设备、工序时间来记录，后续再和包装或成品入库产生的批次关联。
- 一张过程记录不能假设只对应一个成品批次；同一个作业段跨过 24:00 或跨批号时，后续可能关联到两个或多个成品批次。
- 因此过程记录本身先记录“这段时间、这条线、这台设备、这个产品发生了什么”，批次关系后面通过包装/成品入库节点补上。
- 车间成品产出入库表更接近“批次生成或批次确认节点”，这里通常需要生成或关联成品批次。
- 生产计划先作为记录表保留，不升级成完整计划排产系统。
- 过程参数如果有生产填写、质检复核的情况，复核人也从系统员工/用户中选择，不手写姓名。
