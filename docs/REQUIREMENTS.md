> **目标**：让 Claude Code 成为系统全权代理——自主操作业务、发现问题、修复验证，无需人工中转

#### 核心架构决策

MCP 定位为「API 代理 + 发现层」，而非 1:1 映射每个端点。

- **扩展性原则**：新功能上线只需完善 Swagger 描述，MCP 工具集长期稳定（~10个工具覆蓋全部现在和未来功能）
- **双模式认证**：Admin 模式（日常维护）+ 角色模式（模拟业务场景）
- **工具分四类**：发现层 / 执行层 / 运维层 / 测试层

#### 一、整体架构

采用**三层架构**：

| 层次 | 说明 |
|------|------|
| **Layer 1: 通用表单层** | 增强现有 DynamicForm，支持 defaultValue、auto-fill 字段类型 |
| **Layer 2: 专用业务组件层** | AllergenTable.vue（致敏原表）等不可配置化的复杂组件 |
| **Layer 3: 流程编排层** | ProcessTemplate / ProcessInstance，管理多步骤顺序推进 |

**流程步骤**（严格顺序，上一步提交完成后才能进入下一步；仅 Step 7/8 需 HACCP 小组审批）：

```
Step 1: 一. 产品研发与立项申请
Step 2: 二. 设计输入
Step 3: 三. 风险识别              ← 3.1 致敏原表 + 3.2 其他风险识别 + 3.3 食品添加剂
Step 4: 四. 研发试验原始记录（小试）
Step 5: 五. 验证评审（中试）
Step 6: 六. 验证评审
Step 7: 七. 新产品危害评估与验证
Step 8: 八. 开发评审&新产品放行审批
Step 9: 九. 开发输出
```

**新增数据模型（后端）**：

```
ProcessTemplate  — 研发流程模板（固定不变）
ProcessInstance  — 每次具体研发实例，记录当前步骤和状态
ProcessStepData  — 每步填写的数据（Step 7/8 额外含审批记录）
```

#### 二、基础设施改动（所有步骤共用）

**需新增的字段类型（扩展 DynamicForm）**：

| 字段类型 | 说明 |
|---------|------|
| `defaultValue` 属性 | 所有字段支持配置默认值，用户可修改 |
| `auto-username` | 只读，自动填入当前登录用户姓名 |
| `auto-date` | 只读，提交时自动填入当前日期 |
| `section-header` | 非输入字段，仅渲染分组标题（如"产品要求"）|
| `static-content` | 只读内容区，渲染 Markdown/HTML（如协作部门职责、CCP/OPRP 表格）|
| `table-input` | 通用动态表格，可增减行，列数可配置 |
| `approval-step` | HACCP 小组成员的交互式审批节点，审批通过后系统记录审批人姓名和时间戳 |
| `range-select` | 数值范围下拉，配置 min/max/step，如（20-50℃，步长1）|
| `constrained-number` | 数字输入，带强制最小值或范围约束，超出范围时阻止提交 |
| `checkbox-text` | 复合字段：checkbox + 关联文本输入，checkbox 未勾选时文本禁用 |
| `auto-display` | 只读展示，自动从指定步骤的指定字段取值（用于跨步骤数据带出）|
| `template-content` | 固定文字模板，`{fieldName}` 占位符自动替换为同步骤指定字段值，整体只读 |

**新增专用业务组件**：

| 组件 | 用途 |
|------|------|
| `AllergenTable.vue` | 致敏原信息表，集成仓库，含汇总行 |
| `ProcessParams.vue` | 工艺参数组件，含条件显示逻辑（Step 4/5 共用） |
| `DigitRoller.vue` | 个/十/百位独立滚轮拨盘，类似调时间，用于炉温和风机频率输入 |

**新增字段行为：条件显示**：
- 某字段的显示/隐藏取决于另一字段的值
- 例：`使用蛋液` 选"全蛋"时，隐藏"打发机数据-蛋黄"和"打发机数据-蛋清"整个分组
- 在字段配置中增加 `showWhen: { field: 'xxx', value: 'yyy' }` 描述依赖关系

**审批人配置**：按角色动态分配（非固定人员），由系统根据当前用户所在部门的角色关系确定审批人。

##### 5.2 三.2 其他风险识别（内控标准）

- 类型：`static-content`，**直接输出，不可编辑，不做任何修改**
- 固定输出内容：

```
1 感官要求
1.1 色泽：均匀颜色或与添加的原辅料相对应的颜色，均匀一致
1.2 气味：具有该产品应有的气味，无异味
1.3 杂质：无正常视力可见外来异物

2 理化指标
2.1 水分/(g/100g)：≤35（内控标准），检验方法 GB 5009.3

3 微生物指标
3.1 菌落总数：n=5, c=2, m=10⁴, M=10⁵ CFU/g，检验方法 GB 4789.2
3.2 大肠菌群：n=5, c=2, m=10, M=10² CFU/g，检验方法 GB 4789.3
3.3 霉菌：≤150 CFU/g，检验方法 GB 4789.15

4 检验方法
4.1 组织形态、色泽、滋味与口感、杂质按目测法测定。
4.2 理化指标按 GB 5009 规定的方法测定。
4.3 微生物指标按 GB 4789 规定的方法测定。
4.4 产品要求按 GB 7099《糕点面包》规定的方法测定。
4.5 净含量：常温条件下，采用校准后的电子秤称量测定。
4.6 外观检验：
4.6.1 标签：每箱产品均有标签，标签内容应符合 GB 7718 的规定。

5 检验规则
5.1 组批：按同一班次、同一品名、同一规格的产品为一批。
5.2 抽样：
5.2.1 感官及理化指标检验样品的抽样方法按 GB 7099 规定执行，具体如下：
5.2.2 批量范围 < 300 kg 时，抽样数量为 3 kg。
5.2.3 批量范围 300 ≤ n ≤ 800 时，抽样数量为 4~6 件。
5.2.4 批量范围 800 ≤ n ≤ 1000 时，抽样数量为 5% × n。
```
