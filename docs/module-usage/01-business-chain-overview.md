# 业务链总览

## 1. 主业务链

```text
供应商
  -> 物料
  -> 原辅料批次
  -> 来料检验
  -> 仓储库存
  -> 配料执行
  -> 产品批次
  -> 过程/质检/放行
  -> 发货
  -> 客户
  -> 投诉/召回/追溯
```

## 2. 支撑体系

```text
文控
  -> 记录表单索引
  -> 动态表单或业务模块落地
  -> 任务/审批/培训/内审
  -> CAPA/整改
```

## 3. 整改原则

- 业务链优先于当前菜单结构。
- 当前菜单、页面、API 和 Prisma 模型只作为映射层。
- 任何影响追溯的记录必须能回到原辅料批次或产品批次。
- 每个 GAP 必须有编号，后续 PR 必须引用编号。
