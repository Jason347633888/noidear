# Noidear 食品安全 SaaS 上下文

本上下文记录食品安全 SaaS 中主数据、文控、记录表单、生产、配料、库存和追溯相关的业务语言。它用于避免在设计、实现和评审中把同一个现场概念拆成多个平行概念。

本文分两层：

- **Stable Domain Language**：稳定业务术语和已确认领域边界，默认可作为后续讨论口径。
- **Implementation Notes**：当前代码状态快照，必须按标注的核验日期和证据重新确认，不能当作永久事实。

## Stable Domain Language

### 主数据与产品链

**产品档案**:
产品进入系统后的统一主数据，承载产品编号、名称、状态和后续生产引用身份。
_Avoid_: 历史产品表、研发产品表、产品名称事实源

**产品编号**:
系统生成、用户可见的产品业务编号，用于搜索、打印和业务沟通。
_Avoid_: 手填最终编号、另造 productNo/productNumber

**产品名称快照**:
业务记录在发生当时保存的产品名称展示值，用于历史还原，不作为产品事实源。
_Avoid_: 用快照反向维护产品档案

**产品归档**:
产品退出正常业务但保留历史生产、投料、发货、投诉和追溯证据的状态。
_Avoid_: 删除产品、移入回收站

**配方**:
某个产品在一个版本下的物料组成和理论用量。
_Avoid_: Formula、工艺参数表和配方并列使用

**配方明细**:
配方中某一种物料的理论用量、单位、关键性和所属配料区域。
_Avoid_: 在投料记录里重新定义理论配方

**配方归档**:
配方版本退出新生产使用但保留历史解释能力的状态。
_Avoid_: 删除配方、覆盖旧配方

**工序**:
产品或配方下的生产步骤，承载过程控制、CCP 和检验要求的业务位置。
_Avoid_: 把工序当成普通表单章节

### 批次、配料与库存

**产品批次**:
包装机喷码、成品入库、发货、投诉、召回和追溯共同使用的产品批次。
_Avoid_: 生产批次、喷码批次、成品批次并列使用

**产品批次号**:
产品批次的现场编号，来自包装机实际喷码或包装/入库环节确认。
_Avoid_: 另造一套成品批次号

**配料执行**:
某个配料区在某个时间完成的一次配料活动。
_Avoid_: 把一次配料直接叫成产品批次

**投料记录**:
产品批次与原辅料批次之间用于追溯的桥接结果。
_Avoid_: 把投料记录当成配料区现场唯一录入入口

**配料投入池**:
一次或多次配料执行形成的实际投入集合，可归集到一个或多个产品批次。
_Avoid_: 虚构每个产品批次精确分摊的原料重量

**原辅料批次**:
原料、辅料或包材进入库存后的批次，投料时按配料区库存和 FIFO 使用。
_Avoid_: 手填无法关联库存的批次文本

**批次质量状态**:
原辅料批次因质量、过期或锁定控制产生的可用性状态。
_Avoid_: 用库存流向解释质量锁定

**批次库存流向状态**:
原辅料批次在库存流转中的状态，例如在库、已消耗、隔离、处置。
_Avoid_: lot_status、用质量状态替代库存流向

**供应商运营状态**:
供应商主数据是否可被日常业务继续使用的管理状态。
_Avoid_: 用供应商评估状态替代启用/停用

**供应商评估状态**:
供应商经采购或质量评估后的准入状态，独立于供应商是否被运营启用。
_Avoid_: supplier_status、把评估准入状态和运营启停状态合并

**配料区库存**:
某个车间配料区内按原辅料批次记录的现场结存。
_Avoid_: 用仓库总库存代替配料区库存

**库存流水**:
库存对象在位置、数量或状态上发生变化的一次可追溯移动记录。
_Avoid_: 把入库、出库、调拨、盘点调整、成品入库、成品出库拆成互不相干的流水事实源

**班组**:
对某个班次生产、配料、包装或确认动作负责的现场责任组织。
_Avoid_: 只用操作人替代班组责任

**部门**:
系统管理中的组织主数据，用于承载用户归属、部门负责人、权限配置和组织初始化。当前产品口径定义为单层部门，不支持上下级层级关系。
_Avoid_: 树形部门、父子部门事实源

**部门本级权限**:
单层部门口径下授予某个部门自身的权限边界，不包含下级部门继承或扩散语义。
_Avoid_: 子部门隔离、本部门及下级、把下级部门权限退化成本部门权限

**部门负责人**:
某个部门当前的责任用户，由部门正式字段直接指定，用于组织初始化、部门治理和责任归属。负责人事实源是部门对用户的直接指派；角色只决定候选资格，不直接等于负责人身份。
_Avoid_: 仅凭角色推断负责人、从操作记录反推负责人

**系统角色**:
系统治理中用于表达管理员、部门负责人和普通用户身份边界的正式角色主数据。系统角色的正式来源是角色主数据本身，而不是用户身上的影子字符串；运行时不再保留独立影子 role 事实源。
_Avoid_: 用影子字段、临时文本或页面局部约定充当正式角色来源

**运行时角色代码**:
认证后供权限判断消费的系统角色代码，必须来自正式系统角色，不构成新的角色事实源。
_Avoid_: 把 JWT 内部字段名、导入列名或页面展示文本当成正式角色来源

**组织与权限初始化完成**:
系统第一次可用时的阶段性完成状态，表示组织与权限基础骨架已经建立完成。初始化完成后，后续负责人异常、用户未归属部门等问题属于日常治理异常，不会把系统重新打回首次初始化阶段。
_Avoid_: 把日常治理异常重新解释成首次初始化未完成、在正常初始化流程中重置完成状态

**组织治理异常**:
初始化完成之后，部门负责人缺失、负责人失效、用户未归属部门、用户挂在停用部门等组织与权限口径问题。这类问题属于日常治理，不属于首次初始化未完成。
_Avoid_: 用初始化状态描述日常治理异常

**排班**:
某个工作日期和时间班次对应到责任班组、负责人和成员的安排。
_Avoid_: 从操作时间临时猜负责人

**时间班次**:
按企业规则划分的日班、夜班等生产时间段，可跨自然日。
_Avoid_: 直接按自然日切批次责任

**班前盘点**:
开班配料前对配料区、物料和原辅料批次的实物确认。
_Avoid_: 用月度盘点替代开班确认

**班后盘点**:
交班前对配料区、物料和原辅料批次的实物确认。
_Avoid_: 有差异也正常交班

**接班盘点**:
接班方对上一班遗留配料区库存的确认。
_Avoid_: 默认接受上一班账面库存

**盘点差异**:
配料区账面库存和实物库存不一致的异常事实。
_Avoid_: 默认作为普通盘盈盘亏处理

### 追溯与质量闭环

**公司 ID**:
系统中用于隔离不同公司数据的租户边界标识。业务表使用 `company_id` 存储，认证上下文对外暴露为 `companyId`。
_Avoid_: 在业务服务中硬编码 company_id、把操作人 ID 当成公司边界

**追溯主链**:
从原辅料批次经投料关系到产品批次、发货和客户影响范围的正式关系链。
_Avoid_: 用备注、附件或自由文本当正式追溯关系

**正向追溯**:
从上游原辅料批次或产品批次向下游产品、发货和客户影响范围展开的查询。
_Avoid_: 只查当前库存

**反向追溯**:
从发货、投诉、召回或产品批次向上游投料、原辅料批次和供应商展开的查询。
_Avoid_: 只按客户口述批号搜索

**物料平衡**:
围绕原辅料批次或产品批次核对入库、移动、投料、产出、损耗、留样和报废的数量闭环。
_Avoid_: 只看单张投料表判断是否平衡

**追溯演练**:
用真实或模拟问题批次验证系统能否在规定时间内完成正反向追溯和证据汇总的治理活动。
_Avoid_: 普通查询记录

**召回**:
针对已发出产品确定影响范围、通知对象、处置动作和关闭证据的质量事件。
_Avoid_: 普通投诉备注

**不合格**:
产品、物料、过程、设备或文件执行中被判定不满足要求的质量事实。
_Avoid_: 只写在备注里的问题

**整改闭环**:
围绕不合格、投诉、审核发现或变更问题形成的原因分析、措施、验证和关闭过程。
_Avoid_: 只创建待办不验证效果

### 文控与记录表单

**体系文件中心**:
受控文件的统一入口，承载文件库、台账、版本、审批、引用、复审、到期和生命周期。
_Avoid_: 体系文件库和文档台账并列成两个主入口

**受控文件**:
纳入文控生命周期的管理手册、程序文件、作业指导书、公司文件、外来文件或受控附件。
_Avoid_: 普通网盘文件

**文件修订**:
对已发布受控文件发起新版本草稿、审批并发布的换版过程。
_Avoid_: 原地修改已发布文件

**记录表单索引**:
04 记录表单的主索引，用于说明源表单落地到业务模块、动态表单或未落地状态。
_Avoid_: 把 04 当普通文件夹或所有表单的统一填写页

**动态表单**:
用于承接不适合独立业务建模的记录模板和填写记录。
_Avoid_: 用动态表单替代产品、批次、投料、检验等核心业务表

**业务模块承接**:
源记录表单由研发、采购、仓储、生产、品质、设备等模块中的正式业务对象承载。
_Avoid_: 已有业务对象还重复建动态表单事实源

**落地健康状态**:
记录表单索引中用于判断源表单是否已正确关联入口、模板、业务对象和执行证据的状态。
_Avoid_: 只显示有无路由

**引用健康状态**:
体系文件 Markdown 引用是否正常、悬空、无效、冲突或指向旧版本的判断。
_Avoid_: 只展示引用数量

**Excel 导入导出**:
系统中用于批量导入、模板下载、数据预览和统计导出的电子表格交换能力，由后端负责生成和解析。
_Avoid_: 前端本地解析 Excel、浏览器端 xlsx 依赖、页面各自拼装 Excel 文件

**Web/H5 主系统**:
用户当前通过浏览器访问的主业务系统，承载工作台、文控、记录、仓储、质量、设备、追溯和系统治理等页面。
_Avoid_: 把未设计的移动端草稿工程当成主系统 H5 页面

**移动端交付面**:
独立设计给手机 App、小程序、离线采集或现场拍照同步使用的移动产品面。
_Avoid_: 响应式 CSS、浏览器页面的手机适配、Web/H5 主系统

**应用内备份管理**:
在业务 Web/H5 主系统内手动触发数据库或对象存储备份、查看备份历史和删除备份记录的运维页面能力。备份属于部署运维基线，不作为当前业务系统功能保留。
_Avoid_: 在业务应用内用 docker exec 触发备份、把备份历史当业务记录

**应用内健康管理**:
在业务 Web/H5 主系统或业务 API 中查看数据库、缓存、对象存储、磁盘和系统信息的运维能力。它不作为当前业务系统功能保留。
_Avoid_: 把依赖服务健康详情、磁盘信息、系统信息做成业务功能

**运行存活探针**:
供 Docker、Kubernetes、反向代理或部署平台判断服务进程是否存活的最小探针。
_Avoid_: 依赖健康详情、业务健康度、运维大屏

**通用回收站**:
跨文档、记录模板、记录等多个业务对象提供统一查询、恢复、永久删除和自动清理的横向恢复入口。当前不作为业务系统功能保留。
_Avoid_: 用一个通用恢复入口替代各业务对象自己的归档、作废、撤销或删除规则

**通用批量导出中心**:
跨用户、文档、任务、记录、偏差等多个业务对象提供统一导出入口的后台工具页和通用导出 API。当前不作为业务系统功能保留。
_Avoid_: 用一个 `/export/*` 中心替代业务页面里的具体导出动作

### 任务、审批与工作流

**正式审批**:
对业务对象作出同意、驳回、转交或撤销等决策的审批事实。
_Avoid_: 用普通待办或工作流任务替代审批事实

**历史审批记录**:
旧审批表中已经产生的文档审批证据，仅用于历史兼容、审计查询和签名显示。
_Avoid_: 给新文档继续写旧审批记录、把历史兼容路径当成新审批入口

**待办收件箱**:
用户需要处理事项的统一入口投影。
_Avoid_: 把待办当成审批、记录填写或业务执行事实源

**记录填写任务**:
要求用户在指定时间或条件下填写某张记录表单的执行任务。
_Avoid_: 用普通任务替代记录表单填写任务

**周期任务生成器**:
按频率、班次、日期或规则生成记录填写任务或待办的调度机制。
_Avoid_: 把调度规则当成已经发生的执行事实

**工作流编排**:
把多个任务、审批或状态节点串成流程的编排能力。
_Avoid_: 用工作流编排另造第二套审批事实源

## Relationships

- 一个 **产品档案** 可以有多个 **配方** 版本，但新生产只能使用可用产品和可用配方。
- 一个 **配方** 包含一条或多条 **配方明细**。
- 一条 **配方明细** 可以指定一个 **配料区库存** 所属的配料区域。
- **产品归档** 会让相关 **配方** 和 **工序** 退出正常业务，但不改写历史 **产品批次**。
- 一个 **产品批次** 可以归集一个或多个 **配料投入池**。
- 一个 **配料投入池** 可以关联一个或多个 **产品批次**。
- 一个 **配料执行** 包含一条或多条 **原辅料批次** 投入明细。
- **投料记录** 是产品批次与原辅料批次的追溯桥接结果，可由 **配料执行** 归集确认后生成或兼容保留。
- **配料执行** 扣减 **配料区库存**，不是直接扣减仓库总库存。
- **库存流水** 是长期统一库存移动事实源，应覆盖原辅料入库、领料、调拨、退料、报废、盘点调整、成品入库和成品出库。
- 夜班跨零点时，同一班次可以产生两个 **产品批次**，例如 20260429 和 20260430。
- 一个 **排班** 把 **时间班次** 绑定到一个 **班组**。
- **班前盘点** 或 **接班盘点** 有 **盘点差异** 时，不应正常开班配料。
- **班后盘点** 有 **盘点差异** 时，不应正常交班。
- **追溯主链** 以 **原辅料批次**、**配料执行**、**配料投入池** 和 **产品批次** 的关系为核心。
- **正向追溯** 从上游批次展开到下游产品和客户影响范围。
- **反向追溯** 从下游批次、发货、投诉或召回展开到上游原辅料批次和供应商。
- **物料平衡** 使用库存移动、盘点、投料、产出、留样、损耗和报废共同解释数量闭环。
- **整改闭环** 可以来源于 **不合格**、投诉、内审发现、追溯演练或召回。
- **体系文件中心** 管 **受控文件** 的版本生命周期；**记录表单索引** 管 04 记录表单的落地关系。
- 一个 **记录表单索引** 条目可以由 **业务模块承接**，也可以由 **动态表单** 承接。
- **动态表单** 可以记录填写事实，但不替代已经存在的主数据、批次、投料或检验事实源。
- **引用健康状态** 属于受控文件关系；**落地健康状态** 属于记录表单落地关系。
- **Excel 导入导出** 可以服务记录、文控、培训和系统治理等模块，但文件生成和解析事实统一归后端服务。
- **Web/H5 主系统** 可以有移动端响应式显示；这不等于启用了独立 **移动端交付面**。
- **MinIO** 承载业务文件存储；**应用内备份管理** 不承载业务文件事实，剔除它不等于删除文件上传、预览或归档能力。
- **应用内健康管理** 剔除后仍保留 **运行存活探针**；这不影响文控 **引用健康状态**、访客健康状态或员工健康记录等业务语义。
- **通用回收站** 剔除后，业务对象仍可保留 `deletedAt`、归档、作废或撤销等自身状态；是否允许恢复由各业务模块规则决定。
- **通用批量导出中心** 剔除后，业务场景内导出仍可保留；统计、审计日志、追溯、记录 PDF、批次 PDF、培训统计等应由各自业务模块承接。
- **正式审批** 承载审批决策事实；**待办收件箱** 只投影用户需要处理的事项。
- 新文档的 **正式审批** 由统一审批实例承载；**历史审批记录** 只服务旧文档兼容和证据展示。
- **记录填写任务** 是记录表单执行要求；**周期任务生成器** 只负责生成任务，不代表任务已执行。
- **工作流编排** 可以串联任务和审批，但不得另造第二套审批事实源。
- **部门本级权限** 只作用于一个 **部门** 本身；单层部门模型下不存在子部门继承权限。
- **组织与权限初始化完成** 后，**组织治理异常** 只能作为治理问题暴露，不能让系统回退到首次初始化阶段。

## Flagged ambiguities

- "生产批次"、"喷码批次"、"成品批次"曾被混用。已决议：业务文档统一使用 **产品批次**，代码当前由 `ProductionBatch` 承载。
- `FinishedGoodsBatch` 曾被用作成品批次层。已决议：它不再作为核心业务批次概念，业务侧统一使用 **产品批次**。
- "删除产品"曾用于产品退出正常业务。已决议：用户侧统一使用 **产品归档**，历史追溯证据保留。
- "配方"和"Formula"不再并列。已决议：业务文档统一使用 **配方**。
- "体系文件库"和"文档台账"曾作为两个入口并存。已决议：统一收敛为 **体系文件中心** 的不同视图。
- "04 记录表单"曾被误解为普通文件夹或统一填报页。已决议：它是 **记录表单索引**，用于路由到业务模块或动态表单。
- "Excel 导入导出"曾在前端页面各自生成或解析。已决议：统一由后端服务处理，前端不得保留浏览器端 `xlsx` 依赖。
- "mobile"曾同时指移动工程、后端移动 API 和 Web 响应式样式。已决议：当前产品保留 **Web/H5 主系统**，剔除未设计的 **移动端交付面**，但不删除 Web 响应式样式。
- "备份管理"曾作为业务系统页面出现。已决议：剔除 **应用内备份管理**；数据库和对象存储备份由部署运维基线处理，业务系统保留 MinIO 文件存储能力。
- "健康管理"曾同时指运维健康 API、文控引用健康和人员健康状态。已决议：剔除 **应用内健康管理**，保留 **运行存活探针** 和业务健康语义。
- "回收站"曾作为跨业务对象统一恢复入口。已决议：剔除 **通用回收站**；不因此删除业务对象自己的 `deletedAt`、归档、作废或撤销语义。
- "批量导出"曾作为后台通用工具页和多个业务模块内导出混用。已决议：剔除 **通用批量导出中心**；保留业务页面内有真实使用场景的具体导出。
- "盘点差异"不得默认按普通盘盈盘亏跳过。已决议：班前、接班、班后发现差异时走异常处理和确认闭环。
- `approval`、`unified-approval`、`workflow`、`task`、`todo`、`record-task`、`scheduled-task` 曾被混用。已决议：业务口径区分 **正式审批**、**待办收件箱**、**记录填写任务**、**周期任务生成器** 和 **工作流编排**。
